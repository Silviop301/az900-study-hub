Shared Responsibility Model

🔥 cai muito

Quanto mais gerenciado for o serviço, mais responsabilidades passam para a Microsoft. O cliente sempre mantém responsabilidade sobre dados, identidades, contas e acessos.

ModeloClienteMicrosoftLembrete
On-premisesTudo: dados, apps, OS, rede, servidores e datacenter.Nada.Você gerencia tudo.
IaaSDados, identidades, apps, sistema operacional e configurações.Infraestrutura física, armazenamento físico, rede física e datacenter.Você administra a VM.
PaaSDados, identidades, código da aplicação e configurações.OS, runtime, servidores e infraestrutura.Você foca no código.
SaaSDados, contas, permissões e acesso.Aplicação, OS, servidores, rede e datacenter.Você usa o software.

Infraestrutura física

Regions: áreas geográficas com datacenters Azure.

Availability Zones: datacenters separados dentro de uma região; protege contra falha de datacenter.

Region Pairs: regiões pareadas para disaster recovery.

⚠️ Pegadinha: Availability Set não é igual a Availability Zone.

💻 Compute

  • VM: IaaS, controle total.
  • App Service: PaaS para web/API.
  • Functions: serverless por evento.
  • Logic Apps: workflows/conectores.
  • AKS: Kubernetes gerenciado.

🌐 Networking

  • VNet: rede privada no Azure.
  • Peering: conecta VNets.
  • VPN Gateway: criptografado via internet.
  • ExpressRoute: conexão privada dedicada.
  • Load Balancer: distribui tráfego.

🗄️ Storage & DB

  • Blob: objetos/dados não estruturados.
  • Files: compartilhamento SMB.
  • Disk: disco para VM.
  • Cosmos DB: NoSQL global.
  • SQL DB: relacional PaaS.

Heatmap por domínio

A Microsoft pergunta por palavra-chave

Regra mental

Leia o cenário e procure o verbo: controlar acesso, impor regra, estimar custo, monitorar app, automatizar workflow. O verbo quase sempre entrega o serviço.

Não confundaDiferença que cai

Toque/clique para virar. Use para revisão ativa.

SLA
Service Level Agreement: garantia formal de disponibilidade. Ex: 99,9% de uptime.
Composite SLA
SLA combinado de vários serviços. Normalmente multiplica os SLAs: 99,9% × 99,9% ≈ 99,8%.
Availability Zones
Datacenters fisicamente separados dentro de uma região. Protege contra falha de datacenter.
Availability Sets
Distribui VMs em fault/update domains. Protege contra falhas/manutenções dentro do datacenter.
RBAC
Controla quem pode fazer o quê. Palavra-chave: permissões/autorização.
Azure Policy
Impõe regras e conformidade. Palavra-chave: restringir o que pode ser criado/configurado.
Resource Locks
Impede exclusão ou alteração acidental. Use quando a questão falar em proteger contra delete.
Conditional Access
Regras de login. Exige MFA por local, risco, usuário, grupo ou dispositivo.
Microsoft Entra ID
Identidade em nuvem: usuários, grupos, autenticação, SSO e acesso.
Entra Domain Services
AD DS gerenciado no Azure sem implantar controladores de domínio.
Tags
Organizam recursos e custos por projeto, departamento, ambiente ou centro de custo.
Deallocate VM
Para VM fora de uso e interrompe cobrança de compute. Discos e dados continuam existindo.
TCO Calculator
Compara custo on-premises com custo estimado no Azure para migração.
Azure Advisor
Recomenda melhorias em custo, segurança, confiabilidade, performance e excelência operacional.
Azure Monitor
Coleta métricas, logs e alertas de recursos e aplicações.
Application Insights
Monitoramento de aplicações web: performance, falhas, uso e anomalias.
Log Analytics
Consulta logs com KQL. Usado junto com Azure Monitor.
Defender for Cloud
Avalia postura de segurança e recomenda proteções para recursos e workloads.
Microsoft Sentinel
SIEM/SOAR: correlação de eventos, investigação e threat hunting.
ExpressRoute
Conexão privada dedicada entre on-premises e Azure. Não usa internet pública.
VPN Gateway
Conexão criptografada pela internet pública entre redes.
ARM
Camada de gerenciamento do Azure para criar, atualizar e excluir recursos.
Azure CLI
Linha de comando ideal para Bash/Linux e automação multiplataforma.
PowerShell
Automação Microsoft por cmdlets. Muito usado em ambientes Windows.
